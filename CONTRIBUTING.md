# 🤝 Contributing to GetFOSS Catalog

First off, thank you for considering contributing to the GetFOSS Catalog! 
It’s people like you who make the Free and Open Source community so powerful and dynamic. 
Our goal is to create the most comprehensive, high-quality, and up-to-date directory of FOSS software on the internet.

## 🌟 Our Philosophy

This catalog is not just a random list of links. Every piece of software here represents freedom: the freedom to use, study, modify, and distribute. We focus on true Open Source and Free Software alternatives that respect user privacy and reject proprietary lock-in.

## 📜 Criteria for Inclusion

Before submitting a Pull Request, ensure the software meets the following criteria:

1. **True FOSS Only:** The software must be released under an OSI-approved Open Source license or a Free Software Foundation recognized license. We strictly do **not** accept freemium, shareware, or source-available software that restricts user freedoms.
2. **Active or Historically Significant:** The project should be actively maintained, or it must be a historically significant piece of FOSS (e.g., GNU Tools, legacy foundations).
3. **No Duplicates:** Please check the existing list to ensure the software isn't already included under a different category.
4. **Quality over Quantity:** We prefer a curated list of excellent software with detailed descriptions over an exhaustive list of unmaintained projects.

## 🛠️ How to Contribute

### Adding a new software

1. **Fork** this repository.
2. **Edit** the `README.md` file.
3. Find the appropriate category for the software. If no category fits, add it to the `Other` section or suggest a new category in your Pull Request.
4. Insert the software in alphabetical order within the category using the following template:

* [**Software Name**](https://official-website.com) — A brief, punchy description of what it does and why it's great. ([Review on getfoss.org](https://www.getfoss.org/link-to-review-if-exists))

5. If there is already a detailed review on [getfoss.org](https://www.getfoss.org/), please link to it in the review section. If not, you can leave the review link pointing to the main site or omit the review section entirely.
6. **Commit** your changes with a clear message, e.g., `Add [Software Name] to [Category]`.
7. Submit a **Pull Request**.

### Reporting Issues or Suggesting Improvements

If you find a broken link, an inaccurate description, or want to suggest a new category, feel free to [Open an Issue](../../issues). Please provide as much context as possible.

## 💡 A Note on Reviews

When a community member submits a Pull Request for a new FOSS software, the GetFOSS team may decide to write a comprehensive review for it on our main website. By contributing to this list, you help us discover the next great piece of Open Source software!

Thank you for helping the world break free from proprietary software! 🐧
