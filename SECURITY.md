# 🛡️ Security Policy for GetFOSS Catalog

While the GetFOSS Catalog is a curated list of links and descriptions rather than executable software, the security, privacy, and trust of our community remain our highest priorities. 

Because we link to external projects and software, it is crucial that the catalog remains free from malicious, misleading, or compromised entries.

## 📜 Reporting a Security or Trust Issue

If you discover any of the following, please report it to us immediately:

1. **Malware or Compromised Software:** A linked project has been hijacked, or the official download contains malware, spyware, or undisclosed excessive telemetry.
2. **License Misrepresentation:** A software entry is listed as FOSS but is actually proprietary, freeware, source-available, or has restrictive clauses that violate the Open Source Definition or Free Software Definition.
3. **Phishing or Malicious Links:** An official website URL in the catalog has expired and is now redirecting to a phishing site, or the domain has been taken over by malicious actors.
4. **Privacy Violations:** The software engages in undisclosed data harvesting or violates user privacy in ways not aligned with FOSS principles.

## 🚨 How to Report

**Please do not report security vulnerabilities through public GitHub Issues.**

Instead, please report them by [opening a GitHub Issue using the "Security" label](https://github.com/GetFoss/GetFOSS-Catalog/issues/new?labels=security) or by contacting the repository maintainers directly.

To help us resolve the issue quickly, please include:

* The name of the software in question.
* The specific reason for the report (e.g., "Link redirects to malware," "License is not OSI-approved").
* Evidence or proof of the issue (e.g., a link to a VirusTotal report, the specific restrictive license clause, or a screenshot of the malicious redirect).

## ⚙️ Our Commitment

We take all security and trust reports seriously. Upon receiving a valid report:

1. We will review the entry within 24 hours.
2. If the software is found to be malicious, compromised, or misleadingly licensed, the entry will be **immediately removed** from the catalog.
3. If the official website is compromised, the link will be removed or replaced with a safe alternative (like the source code repository) until the official site is restored.

Thank you for helping keep the GetFOSS Catalog safe, transparent, and trustworthy for everyone. 🐧
